# ФинКратко — бесплатный бот конспектов YouTube → Telegram

Проект делает автоматические конспекты новых роликов с YouTube-каналов по финансам и публикует их в Telegram.

## Что делает бот

1. Находит новое видео по списку каналов.
2. Скачивает аудио с YouTube.
3. Расшифровывает его локально через `faster-whisper`.
4. Делит выпуск на смысловые блоки.
5. Извлекает факты.
6. Отбрасывает непонятные, оборванные и шумные фразы.
7. Мягко переформулирует факты.
8. Публикует пост в Telegram.

## Формат поста

- Вверху: `Канал — ...`
- Ниже: кликабельное название видео
- Затем: компактные тезисы
- Внизу: **обычная ссылка на YouTube**, чтобы Telegram подтягивал превью/обложку
- В самом низу: кликабельный текст `ФинКратко`

## Бесплатный запуск

### Вариант 1 — GitHub Actions

Подходит как бесплатный MVP.

Что нужно сделать:

1. Создать репозиторий на GitHub.
2. Залить в него все файлы этого проекта.
3. В `Settings -> Secrets and variables -> Actions` добавить secrets:
   - `YOUTUBE_API_KEY`
   - `TELEGRAM_BOT_TOKEN`
   - `TELEGRAM_CHAT_ID`
   - `OWN_CHANNEL_URL` = `https://t.me/fincratko`
4. В `Variables` добавить:
   - `CHANNEL_HANDLES` = `@finam_invest,@brokerru,@rbcinvest,@dengi_ne_spyat`
5. Убедиться, что бот является администратором Telegram-канала.
6. Запустить workflow вручную через **Actions -> Run workflow**.

### Вариант 2 — локально

```bash
pip install -r requirements.txt
export YOUTUBE_API_KEY="..."
export TELEGRAM_BOT_TOKEN="..."
export TELEGRAM_CHAT_ID="..."
export OWN_CHANNEL_URL="https://t.me/fincratko"
python -m app.main
```

## Временное окно публикации

В коде задано окно публикации по Москве: `08:00–21:00`.

- если видео обработано внутри окна — бот публикует сразу;
- если видео обработано после 21:00 МСК — пост кладется в `data/pending_queue.json` и выходит на следующем запуске в разрешенное время.

## Где улучшать качество

### `data/normalization_rules.json`
Сюда можно добавлять:
- исправления терминов;
- алиасы эмитентов;
- доменные замены (`бренд -> Brent`, `офз -> ОФЗ` и т.д.).

### `data/feedback.jsonl`
Сюда можно складывать плохие и хорошие формулировки для будущего дообучения.

Пример строки:

```json
{"video_id":"abc123","bad_text":"дивиденд 10,2 рубля на акцию","good_text":"Татнефть: дивиденд за квартал может составить 10,2 рубля на акцию","reason":"metric_without_subject"}
```

### `debug_reports/*.json`
Там сохраняются:
- принятые факты;
- отклоненные кандидаты;
- причины отклонения.

Это лучший способ понять, почему конкретная мысль не попала в итог.

## Что можно улучшить потом
